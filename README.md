# Raylib - Jolt and Networking template

This template combines [Raylib](https://www.raylib.com/) with [Networking](github.com/nathhB/nbnet) and a [Physics Engine](https://github.com/jrouwe/joltphysics) (with C bindings supplied by [JoltC](https://github.com/SecondHalfGames/JoltC/)). It should help you get a multiplayer physics-based game going pretty quickly.

It's intended to be used as a base for building another simple multiplayer network game with Physics, so you don't need to spend so much time digging through build systems and binding code.

## Bits and Pieces

The default split creates two binaries, a [client](src/client.c) and [server](src/server.c). The *client* is a thin renderer for an *authoritative* server. Physics packets are delta-compressed with a lowest-three encoding scheme for Quaternions (I implemented all this loosely based on this [Gaffer on Games](https://gafferongames.com/post/snapshot_compression/) series of blog posts). By default, the client simulates **3 ticks behind** the server. This can all be configured in the [message](src/message.h) header, which describes the network protocol.

The server maintains state and operates on entites using an extremely simple [ecs](src/ecs.h) which you can use as a starting point.

I've added a simple [Orbit camera](src/orbit_camera.c) and [character controller](src/ecs.c#L120) to get you started.
